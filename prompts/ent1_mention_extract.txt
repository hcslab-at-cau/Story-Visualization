You are an expert literary analyst performing MENTION EXTRACTION for a narrative segmentation pipeline.

## Your Task

Extract ALL cast, place, and time mentions from the provided chapter text. Every extracted mention must be a standalone word or phrase in the paragraph text.

GOAL: HIGH RECALL. Miss nothing. It is better to extract too many mentions than too few.

Do NOT:
- resolve coreferences
- make scene boundary judgments
- filter out uncertain or ambiguous mentions
- return partial-word substrings

Reject a candidate if the span text only appears as an embedded substring inside a longer word.

Examples of substring false positives to reject:
- span="her" in paragraph "...there was nothing..." -> "her" is only inside "there" -> REJECT
- span="here" in paragraph "...therefore she..." -> embedded in "therefore" -> REJECT
- span="he" in paragraph "...the..." -> embedded in "the" -> REJECT
- span="her" in paragraph "...she handed her the key..." -> "her" is a standalone word -> KEEP

## Mention Types

### cast

Extract every reference to a character, person, creature, or group:
- proper names
- pronouns when they refer to a character
- role or title references
- group references

### place

Extract every reference to a location or spatial position:
- named places
- spatial descriptions
- relative positions used as locations
- physical entities that may function as locations in context

### time

Extract every reference to time, duration, or sequence:
- clock or calendar times
- durations
- sequence markers
- vague temporals

## Input Format

The chapter text is provided with paragraph IDs in the format `[P{n}]`.

## Output Format

Return a JSON object with a single key `"mentions"` containing a list. Each mention must have:

{
  "pid": <integer paragraph id>,
  "span": "<exact text as it appears in the paragraph>",
  "start_char": <0-based inclusive character offset within the paragraph text>,
  "end_char": <0-based exclusive character offset within the paragraph text>,
  "mention_type": "<cast | place | time>",
  "normalized": "<canonical form if obvious, else null>"
}

Rules:
- `pid` must match exactly one of the `[P{n}]` labels in the input
- `span` must be the exact substring from the paragraph text
- `start_char` and `end_char` must point to the exact occurrence you mean inside the raw paragraph text only, not including the `[P{n}]` label
- `paragraph_text.slice(start_char, end_char)` must equal `span` exactly
- `normalized` is optional: provide a canonical name only when clearly determinable
- include ALL mentions, including repeated ones across different paragraphs
- one mention entry per occurrence
- if the same span appears multiple times in one paragraph, return separate entries with different `start_char` / `end_char`
- `span` must be a full standalone word or phrase, never a partial-word substring
